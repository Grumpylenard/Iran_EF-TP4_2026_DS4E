# Web Scraped Incidents

This directory contains incident data scraped from CENTCOM and CTP-ISW sources.

## Data Source

Generated by: `05_web_scrape_incidents.R`

## File

- `incidents.csv` - Standalone CSV with strike/incident locations

## Schema

| Column | Type | Description |
|--------|------|-------------|
| `lat` | double | Latitude of incident location (may be NA if geocoding failed) |
| `lon` | double | Longitude of incident location (may be NA if geocoding failed) |
| `date` | character | Incident date from source report |
| `location_name` | character | Extracted location name (city, province, or description) |
| `target_type` | character | Type of target (e.g., missile base, radar site, etc.) |
| `source` | character | Data source: "CENTCOM" or "CTP-ISW" |
| `report_url` | character | URL to original source report |

## Usage

To regenerate the data:

```bash
# With default settings (10 CENTCOM pages, 5 ISW pages)
Rscript 05_web_scrape_incidents.R

# With custom page limits
CENTCOM_PAGES=20 ISW_PAGES=10 Rscript 05_web_scrape_incidents.R

# Preview mode (no file output)
WRITE_OUTPUT=FALSE Rscript 05_web_scrape_incidents.R

# Disable a source
SCRAPE_CENTCOM=FALSE Rscript 05_web_scrape_incidents.R
```

## Notes

- Extraction uses regex patterns only (no LLM dependency)
- Geocoding via OpenStreetMap Nominatim API
- Network access required for scraping and geocoding
- Respects rate limits with 0.8s delay between requests
- Rows without coordinates are included but have NA lat/lon values
